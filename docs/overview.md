# Mood-based Recipe Recommendation App

## Overview

This repository contains the full stack codebase implementing features from the SCRUM Jira issues list. The app is a mood-based recipe recommendation platform with the following features:

### MVP Features (SCRUM-288 to SCRUM-293)
- React SPA frontend with login, mood input, recipe display, and favorites management
- Backend Node.js/Express API with secure JWT authentication and bcrypt password hashing
- MongoDB data models for Users, Recipes, Moods, UserMoodEntries
- API endpoints for user registration, login, mood submissions, recipe retrieval by mood, and favorites management
- Security measures including HTTPS enforcement, data privacy compliance, and user data deletion support
- Performance optimizations including scalable backend setup, caching, and monitoring
- Logging and monitoring setup with ELK, Prometheus, Grafana

### Post-MVP Features (SCRUM-294 to SCRUM-296)
- Sentiment analysis integration for advanced mood detection
- Personalized recommendation engine leveraging user preferences and history
- Extended data models for UserPreferences and Reviews
- Recipe nutritional info and dietary filters in API and UI
- Social sharing and community review features with moderation awareness

### Future Expansion (SCRUM-297 to SCRUM-298)
- AI-driven personalized recipe recommendations with ML models and pipeline
- Multilingual support and user-generated recipe submissions with moderation

## Repository Structure

- `backend/` - Express backend including models, routes, and app server setup
- `frontend/` - React SPA frontend source
- `docs/` - Documentation including this overview
